JavaScript的异步编程

发表于 2018-12-09


  上文介绍到了事件循环和Promise,本来打算就此为止的，但是后来想了，既然谈到了异步编程，也想将Js异步的最终实现async也梳理下，因此推出这篇js的同步编写异步代码介绍。

前言

如上文所言，js为了简单的处理Dom操作、页面交互，因此设计成了单线程的语言。然而js是单线程的，浏览器却不是单线程的，因此，我们可以用其他线程来处理主线程（即JS编译线程，本文统一为主线程）不愿意浪费时间等待的异步操作，并且监听消息队列，将回调函数放入主线程。但是正常异步编程会行程层层嵌套的糖葫芦串代码，因此我们来看看如何更优雅的写出异步代码，下面放出个优雅的异步控制代码（你可以想想用callBack来写是什么样子的）

迭代器Iterator

我们先来巩固下基础知识，ES6中新增了Map和Set，加上原有的Array和Object，Js现在一共用4中数据集合。对于各个数据的遍历和使用，需要一个统一的接口来处理，因此，ES6推出了Iterator和for…of

接口信息

Iterator接口默认部署再数据结构Symbol.iterator属性上。因此，只要有Symbol.iterator属性，该数据结构就是可迭代的。以下就是基本迭代功能的数据结构：

  • Array
  • Map
  • Set
  • String
  • TypedArray
  • 函数的 arguments 对象
  • NodeList 对象

默认场景

for…of 会默认调用Iterator 接口，以下还有几种场景也会调用

  1. 解构和拓展运算符

  2. yield*
    此处下文讲

  3. 数组遍历
    数组遍历会调用iterator接口，因此，能接收数组的场景，都调用了iterator，如：

  • for…of
  • Array.from()
  • Map(), Set()
  • Promise.all()
  • Promise.race()

生成器Generator

那异步编程和我们的迭代器有什么关系了？因为JS单线程的原因，我们对异步编程优化的方向一直是向着同步的趋势。如开篇代码，我们不用再关注层层嵌套的回调来绕来绕去，我们也不希望有写一大堆Promise然后then、catch下去。因此ES6推出了Generator异步编程。

状态机函数

Generator其实是一个状态机，内部封装了多种状态，所有的过程状态都通过yield来定义，最终状态通过return定义，如下：

当我们每次调用next接口时就会返回下一个状态

其实Generator就是迭代器Iterator的函数实现，Iterator不部署在无序的输出的对象(Object)上，因此Genrator通过yield来暂停函数，保证有序的输出序列。Generator通过即可定义，其中可以在类型声明和函数名的任意位置，因为Generator本身是一个高阶函数，因此我习惯于函数名前加*以表明Generator的身份。

yield和next

yield在generator中代表着暂停标志，generator会返回yield后紧跟的值作为返回值。yield本身没有返回值，但是可以通过next传参定义上一个yield值。

第一次执行next： 返回5
第二次执行next： 1 = yield x = y，返回 y + 1 ，2
第三次执行next： 2 = yield(y + 1) = z, 此时，x = 5， y = 1，返回 8

基于Generator的异步编程

现在，我们要抛弃一大堆then的Promise异步编程，让我们的异步代码更优雅。大多数异步编程都是协程完成，即由多个协程共同工作完成：

  1. 协程A开始。
  2. 协程A任务暂停，转接协程B任务，协程B任务开始。
  3. 协程B完成任务，转接协程A。
  4. 协程A恢复执行

Generator就是来完成Js的协程编程

async 函数

上面就是我们的异步编程最终处理吗？是的，在本文来说，是的，原理上。现在我们又有新的问题了，是不是我们每次异步编程，都需要声明一个run，并执行它？ES7中针对这种情况，特意退出了async。

含义

async之于Generator，就像是Class之于Js原型操作。前者都是后者的语法糖，为了进一步优化代码，封装底层原理。我们先来看看以上示例代码的优化

step还是一个标准的Promise异步函数，然而后面我们却少了个run自执行器，*和yield声明方式也变成了async和await，这就是async异步编程的最终优化：

  1. 内置执行器
    async内部自有自执行器，每次等待返回后，自行执行代码至return

  2. 更清楚的语义化
    相比于*和yield声明方式，async和await更容易让人明白这个高阶函数

  3. Promise返回值
    相比于原来的自执行返回的对象，async返回Promise，你完全可以用then进行下一步操作。

基础语法

返回处理

  1. async return返回值会成为then方法的入参
  2. async catch返回会使返回对象Promise变为reject并被catch方法接收
  3. await 异步操作报错时，asncy的返回对象Promise可以通过catch捕获

await命令

  1. await默认应该返回Promise对象
  2. 若果不是Promise则返回对应值
  3. 特殊情况：如果返回的对象包含了then方法，await会将其等同Promise
